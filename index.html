Structure: HTML

Tags

HTML elements are enclosed within tags.

<p>My cat is very grumpy.</p>

My cat is very grumpy.

Tag names are case-insensitive.

<p>My cat is very grumpy.</P>

My cat is very grumpy.

Tags can be nested.

<p>My cat is <em>very</em> grumpy.</p>

My cat is very grumpy.

Some tags are void and don't enclose anything. They stand on their own and don't need a corresponding closing tag.

<p>My cat<br />is very grumpy.</p>

My cat
is very grumpy.

Attributes

Elements can also have attributes.

<p style="color: darkred">My cat is very grumpy.</p>

My cat is very grumpy.

Attributes with only one value don't require the value to be specified. They are called boolean attributes (either they exist or they don't).

<input type="text" disabled="disabled"/>
<input type="text" disabled/>
<input type="text"/>

Comments

Content enclosed within <!-- and --> is ignored by the browser.

<!-- I am a comment! Yayaya -->

Elements

Basic

The most important HTML elements are:

  • <!DOCTYPE html />: a historical artifact that needs to be included in every HTML page.
  • <html>: the root element, the one that contains all other elements.
  • <head>: a container for everything that isn't the content that displays on the page.
  • <body>: a container for all the content that displays on the page – text, images, videos, games, playable audio tracks, etc.

Head

Common elements nested in the <head> are:

  • <title>: the title that appears in the page's browser tab and that is used to describe the page when it is bookmarked.
  • <style>: an internal stylesheet for the page.
  • <link />: a link to an external stylesheet for the page.
  • <script>: a script (or a link to one) that will run in tandem with the page.
  • <base />: the base URL to use for all relative URLs in a document. Learn more about base.
  • <meta />: description of metadata (data about the page) not captured by the previous elements. Learn more about meta and about the meta viewport.

Website Structure

Elements used to represent the website's structure are:

  • <header>: a group of introductory content. If it is a child of <body> it defines the global header of a webpage, but if it's a child of an <article> or <section> it defines a specific header for that section (try not to confuse this with titles and headings).
  • <nav>: the main navigation functionality for the page (home, posts, projects, contacts, search).
  • <main>: the main content unique to this page. Use <main> only once per page, and put it directly inside <body>. Do not nest within other elements.
  • <article>: a block of related content that makes sense on its own without the rest of the page (e.g., a single blog post).
  • <section>: similar to <article>, but it is more for grouping together a single part of the page that constitutes one single piece of functionality (e.g., a mini map, or a set of article headlines and summaries), or a theme. It's considered best practice to begin each section with a heading; also note that you can break <article>s up into different <section>s, or <section>s up into different <article>s, depending on the context.
  • <aside>: content that is not directly related to the main content but can provide additional information indirectly related to it (e.g., glossary entries, author biography, related links). Often placed inside <main>.
  • <footer>: a group of end content for a page.

General text formatting

General text-related elements are:

  • <h1>, <h2> ... <h6> (heading): headings 1 through 6.
  • <p> (paragraph): a paragraph of text.
  • <br /> (break): a newline, or line break.
  • <hr /> (horizontal): a horizontal like, signifying a thematic break.
  • <em> (emphasized): text that has stress emphasis – typically rendered as italic text.
  • <strong> (strong): text that has strong importance, seriousness, or urgency – typically rendered as bold text.
  • <i> (idiomatic): text that is set off from the normal text for some reason (idiomatic text, technical terms, taxonomical designations, etc).
  • <s> (striked): text that is no longer accurate – typically rendered as striked text. Do not use decoratively, instead use CSS.
  • <u> (underline): text that has a non-textual annotation, like an underline. Do not use decoratively, instead use CSS.
  • <blockquote> (blockquote): an extended quotation – typically rendered with indentation.
  • <q> (quote): a short inline quotation – typically rendered as enclosed in quotation marks.
  • <cite> (cite): the title of a cited creative work.
  • <abbr> (abbreviation): an abbreviation or acronym.
  • <address> (address): contact info.
  • <time> (time): either a time on a 24-hour clock, a date in the calendar or a time duration.
  • <small> (small): small print, copyright or legal text.
  • <ins> (inserted): text that has been inserted.
  • <del> (deleted): text that has been deleted.
  • <mark> (marked): text that has been highlighted.

Lists

Elements used to define lists are:

  • <ul> (unordered list): an unordered list of items – typically rendered as a bulleted list.
  • <ol> (ordered list): an ordered list of items – typically rendered as a numbered list.
  • <li> (list item): a list item.
  • <dl> (description list): a list of groups of terms and related descriptions.
  • <dt> (description term): a term in a description list.
  • <dd> (description details): the value for the preceding term in a description list.
  • <menu> (menu): same as ul, but it is used for interactive lists, like lists of buttons.

To nest lists, place a list inside one of the parent's list items.

<ul>
  <li>A</li>
  <li>B
    <ul>
      <li>B.a</li>
      <li>B.b</li>
    </ul>
  </li>
  <li>C</li>
</ul>
  • A
  • B
    • B.a
    • B.b
  • C

Code

Some tags are dedicated to formatting code. They are usually rendered using a monospace font.

  • <code> (code): A generic piece of computer code, like int main(void) { }.
  • <pre> (preformatted): Paragraph that retains extra whitespace that the browser would generally ignore.
  • <var> (variable): A variable name, like result.
  • <kbd> (keyboard): Keyboard and other types of input entered into the computer, like Ctrl+C.
  • <samp> (sample): The output of a computer program, like Hello, World!.

Content (text, images, etc.) enclosed into <a> tags becomes a hyperlink. It can be a link to a different page:

Images

To embed an image into a webpage, use the void <img> tag. Use the src attribute to specify the absolute or relative URL of the image. Give images descriptive filenames as they are counted towards SEO.

<img src="images/black-star-128.png" alt="A black star on a transparent background." width="128" height="128" title="It's a star!" />

The alt attribute specifies text that will be displayed when the image can't be loaded, and/or will be read out loud by screen readers.

<img src="images/black-star-127.png" alt="A black star on a transparent background." width="128" height="128" title="It's a star!" />

There are many reasons for adding alt text:

  • The user is using a screen reader to read the web out to them.
  • The spelling of the file or path name might be wrong.
  • The browser doesn't support the image type.
  • Search engines can match alt text with search queries.
  • Users have turned off images to reduce data transfer volume and distractions.

The content of the alt text depends the purpose of the image.

  • Decoration: you should use CSS background-images for decorative images, but if you must use HTML, add a blank alt="" so that screen readers don't waste time reading it.
  • Content: if your image provides significant information, provide the same information in a brief alt text – or leave it blank if the image is described adequately by the main text body. Don't write redundant alt text.
  • Link: if you put an image inside <a> tags you still must provide accessible link text. In such cases you may, either, write it inside the same <a> element, or inside the image's alt attribute – whichever works best in your case.
  • Text: you should not put your text into images. However, If you really can't avoid doing it, supply the text inside the alt attribute too.

For more information, see Text Alternatives on MDN.

The width and height attributes represent the image's width and height in pixels.

You should specify the actual size of your source images, not the size you want them to appear on the website. If you do need to alter an image's size, either change the actual size of the source image or use CSS.
The HTML for your page and the image are separate resources, fetched by the browser as separate HTTP(S) requests. As soon as the browser has received the HTML, it will start to display it to the user. If the images haven't yet been received (and this will often be the case, as image file sizes are often much larger than HTML files), then the browser will render only the HTML, and will update the page with the image as soon as it is received (causing the rest of the content to move).
By specifying the width and height of the image in advance, the browser will reserve an appropriate amount of space for them and won't cause the rest of the content to be pushed away.

The title attribute displays a tooltip on mouse hover. It has a number of accessibility problems, and it is better to include supporting information in the main article text.

Videos and audio clips

To embed a video into a webpage, use the <video> tag. Use the src attribute to specify the absolute or relative URL of the video, and the controls boolean attribute to display start, pause and similar controls.

<video src="videos/pong.mp4" controls>
  <p>
    Your browser doesn't support HTML video. Here is a
    <a href="videos/pong.mp4">link to the video</a> instead.
  </p>
</video>

The following are all the attributes of the <video> element:

  • src: Absolute or relative URL of the video.
  • controls: Include the browser's default control interface (boolean).
  • width, height: Width and height of the source video.
  • autoplay: makes the video start as soon as it is loaded (boolean). Do not use, it's very annoying.
  • loop: makes the video start again when it finishes (boolean).
  • muted: turns off the audio by default (boolean).
  • preload: Used for buffering large files; it can take one of three values:
    • "none": does not buffer the file
    • "auto": buffers the media file
    • "metadata": buffers only the metadata for the file
  • poster: The URL of an image which will be displayed before the video is played.

If you don't add the controls attribute, build a custom interface using the appropriate JavaScript API. At a minimum, the interface must include a way to start and stop the media, and to adjust the volume.

The <audio> element is exactly like the <video> element, except that it doesn't have the width, height and poster attributes.

Other embeds

Use the <iframe> element to embed other web pages into the current one.

Captions

Add captions to any independent unit of content (an image, a gallery of images, a code snippet, audio, video, equations, a table, etc.) by using the <figure> and <figcaption> elements.

<figure>
  <img src="images/black-star-128.png" alt="A black star on a transparent background." width="128" height="128" />
  <figcaption>It's a star!</figcaption>
</figure>

Tables

Use a <table> element to describe a table. Do not use HTML tables as a layout tool, but only to display tabular data.

To add a single cell to a table, use the <td> (table-data) element. Put the table's data inside the <tbody> (table-body) element.

<table>
  <tbody>
    <tr><td>Hi, I'm your first cell.</td></tr>
  <tbody>
</table>
Hi, I'm your first cell.

Multiple <td> elements are automatically arranged in a row.

<table>
  <tbody>
    <tr>
      <td>Hi, I'm your first cell.</td>
      <td>I'm your second cell.</td>
      <td>I'm your third cell.</td>
      <td>I'm your fourth cell.</td>
    </tr>
  </tbody>
</table>
Hi, I'm your first cell. I'm your second cell. I'm your third cell. I'm your fourth cell.

To create multiple rows, wrap each row in a separate <tr> (table-row) element.

<table>
  <tbody>
    <tr>
      <td>First row, first cell.</td>
      <td>First row, second cell.</td>
      <td>First row, third cell.</td>
      <td>First row, fourth cell.</td>
    </tr>
    <tr>
      <td>Second row, first cell.</td>
      <td>Second row, second cell.</td>
      <td>Second row, third cell.</td>
      <td>Second row, fourth cell.</td>
    </tr>
  </tbody>
</table>

Semantic math elements

Every text in MathML formulas must be included inside these container elements, called token elements.

  • mn: Number literal (2, 3.141, 0x0045, ...)
  • mo: Operator or other punctuation sign (+, ∑, [, ...)
  • mi: Identifier (x, cos, π, ...)
  • mtext: Other text (if, such as, not, ...)

Single-character <mi> elements are automatically italicized. To prevent this, use the mathvariant attribute.

<mi>sin</mi>                      <!-- Multi-character identifier, not italicized -->
<mtext>&nbsp;</mtext>
<mi>x</mi>                        <!-- Single-character identifier, italicized -->
<mtext>&nbsp</mtext>
<mi mathvariant="normal">x</mi>   <!-- Single-character identifier, explicitly not italicized -->

sin   x   x

The browser will look up any operator enclosed in <mo> in an operator dictionary so that the necessary spacing or stretching between surrounding elements is respected.

| 1 x |  ,  u + w →

Common errors include:

  • Forgetting to close an open tag
  • Forgetting to close the quotes on an attribute
  • Forgetting to quote an attribute value containing spaces

HTML performance

  • Prefer .svg to other image formats when possible. Use jpg for photos and png for screenshots and logos.
  • Use responsive images with the "secret" and "size" attributes.
  • Use Lazy Loading. Use attribute loading="lazy"
  • Prefer smaller files across all categories - images, fonts, videos, etc.   Page size should be under 1.5-2MB.
  • Reduce the number of HTTP requests. There are several techniques by which we can minimize the number of requests like combining the files, merging a multiple number of scripts to one, using CSS sprites and image maps, etc.
  • Reduce the number of nodes in the DOM - ideally under 1400 total nodes, better if under 800 total nodes.
  • Minify HTML files - that is, remove all comments and unnecessary spaces so that they load faster.
  • Clean up your media library by deleting unnecessary images and other media files for faster average page load times.
