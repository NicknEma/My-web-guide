HTML

Tags

HTML elements are enclosed within tags.

<p>My cat is very grumpy.</p>

My cat is very grumpy.

Tags can be nested.

<p>My cat is <em>very</em> grumpy.</p>

My cat is very grumpy.

Some tags are void and don't enclose anything. They stand on their own and don't need a corresponding closing tag.

<p>My cat<br />is very grumpy.</p>

My cat
is very grumpy.

Attributes

Elements can also have attributes.

<p style="color: red">My cat is very grumpy.</p>

My cat is very grumpy.

Attributes with only one value don't require the value to be specified. They are called boolean attributes (either they exist or they don't).

<input type="text" disabled="disabled"/>
<input type="text" disabled/>
<input type="text"/>

Comments

Text enclosed within <!-- and --> is ignored by the browser. 

<!-- I am a comment! Yayaya -->

Elements

Basic

The most important HTML elements are:

  • <!DOCTYPE html />: a historical artifact that needs to be included in every html page.
  • <html>: the root element, the one that contains all other elements.
  • <head>: a container for everything you want to include on the HTML page that isn't the content the page will show to viewers. This includes keywords and a page description that would appear in search results, CSS to style content, character set declarations, and more.
  • <body>: a container for all the content that displays on the page, including text, images, videos, games, playable audio tracks, or whatever else.

Head

Common elements nested in the <head> are:

  • <title>: the title that appears in the page's browser tab and that is used to describe the page when it is bookmarked.
  • <style>: an internal stylesheet for the page.
  • <link />: a link to an external stylesheet for the page.
  • <script>: a script (or a link to one) that will run in tandem with the page.
  • <base />: the base URL to use for all relative URLs in a document. See more.
  • <meta />: description of metadata (data about the page) not captured by the previous elements. See more.

Website Structure

HTML provides dedicated tags that you can use to represent different page sections, for example:

  • <header>: a group of introductory content. If it is a child of <body> it defines the global header of a webpage, but if it's a child of an <article> or <section> it defines a specific header for that section (try not to confuse this with titles and headings).
  • <nav>: the main navigation functionality for the page (home, posts, projects, contacts, search).
  • <main>: the main content unique to this page. Use <main> only once per page, and put it directly inside <body>. Do not nest within other elements.
  • <article>: a block of related content that makes sense on its own without the rest of the page (e.g., a single blog post).
  • <section>: similar to <article>, but it is more for grouping together a single part of the page that constitutes one single piece of functionality (e.g., a mini map, or a set of article headlines and summaries), or a theme. It's considered best practice to begin each section with a heading; also note that you can break <article>s up into different <section>s, or <section>s up into different <article>s, depending on the context.
  • <aside>: content that is not directly related to the main content but can provide additional information indirectly related to it (e.g., glossary entries, author biography, related links). Often placed inside <main>.
  • <footer>: a group of end content for a page.

Text formatting

Text-related elements are:

  • <p> (paragraph): a paragraph of text.
  • <h1>, <h2> ... <h6> (heading): headings 1 through 6.
  • <br /> (break): a newline, or line break.
  • <ul> (unordered list): an unordered list of items – typically rendered as a bulleted list.
  • <ol> (ordered list): an ordered list of items – typically rendered as a numbered list.
  • <li> (list item): a list item.
  • <em> (emphasized): content that has stress emphasis – typically rendered as italic text.
  • <strong> (strong): content that has strong importance, seriousness, or urgency – typically rendered as bold text.
  • <dl> (description list): a list of groups of terms and related descriptions.
  • <dt> (description term): a term in a description list.
  • <dd> (description details): the value for the preceding term in a description list.
  • <blockquote> (blockquote): an extended quotation – typically rendered with indentation.
  • <q> (quote): a short inline quotation – typically rendered as enclosed in quotation marks.
  • <cite> (cite): the title of a cited creative work.
  • <abbr> (abbreviation): an abbreviation or acronym.
  • <address> (address): contact info.
  • <time> (time): either a time on a 24-hour clock, a date in the calendar or a time duration.

Code

Some tags are dedicated to formatting code.

  • <code>: For marking up generic pieces of computer code.
  • <pre> (preformatted): For retaining extra whitespace that the browser would generally ignore.
  • <var> (variable): For marking up variable names.
  • <kbd> (keyboard): For marking up keyboard and other types of input entered into the computer.
  • <samp> (sample): For marking up the output of a computer program.

Images

To embed an image into a webpage, use the void <img> tag. Use the src attribute to specify the absolute or relative URL of the image.

<img src="face.png" alt="A yellow smile on a gray background." width="500" height="500" title="Smile!" />

Search engines read image filenames and count them towards SEO; you should give your image a descriptive filename.

The alt attribute specifies text that will be displayed when the image can't be loaded, and/or will be read out loud by screen readers.

<img src="facee.png" alt="A yellow smile on a gray background." width="500" height="500" title="Smile!" />

There are many reasons for adding alt text:

  • The user is using a screen reader to read the web out to them.
  • The spelling of the file or path name might be wrong.
  • The browser doesn't support the image type.
  • Search engines can match alt text with search queries.
  • Users have turned off images to reduce data transfer volume and distractions.

The content of the alt text depends the purpose of the image.

  • Decoration: you should use CSS background-images for decorative images, but if you must use HTML, add a blank alt="" so that screen readers don't waste time reading it.
  • Content: if your image provides significant information, provide the same information in a brief alt text – or leave it blank if the image is described adequately by the main text body. Don't write redundant alt text.
  • Link: if you put an image inside <a> tags you still must provide accessible link text. In such cases you may, either, write it inside the same <a> element, or inside the image's alt attribute – whichever works best in your case.
  • Text: you should not put your text into images. However, If you really can't avoid doing it, supply the text inside the alt attribute too.

For more information, see Text Alternatives on MDN.

The width and height attributes represent the image's width and height in pixels. You should specify the actual size of your source images, not the size you want them to appear on the website. If you do need to alter an image's size, either change the actual size of the source image or use CSS.

The HTML for your page and the image are separate resources, fetched by the browser as separate HTTP(S) requests. As soon as the browser has received the HTML, it will start to display it to the user. If the images haven't yet been received (and this will often be the case, as image file sizes are often much larger than HTML files), then the browser will render only the HTML, and will update the page with the image as soon as it is received (causing the rest of the content to move). By specifying the width and height of the image in advance, the browser will reserve an appropriate amount of space for them and won't cause the rest of the content to be pushed away.

The title attribute gives us a tooltip on mouse hover. It has a number of accessibility problems, and it is better to include supporting information in the main article text, rather than attached to the image in this way.

Videos and audio clips

To embed a video into a webpage, use the <video> tag. Use the src attribute to specify the absolute or relative URL of the video, and the controls boolean attribute to display start, pause, volume and similar controls.

<video src="pong.mp4" controls>
  <p>
    Your browser doesn't support HTML video. Here is a
    <a href="pong.mp4">link to the video</a> instead.
  </p>
</video>

The following are all the attributes of the <video> element.

  • src: Absolute or relative URL of the video.
  • controls: Include the browser's default control interface.
  • width and height: Width and height of the video.
  • autoplay: Boolean attribute that makes the video start as soon as it is loaded. Do not use, it's very annoying.
  • loop: Boolean attribute that makes the video start again when it finishes.
  • muted: Boolean attribute that turns off the audio by default.
  • preload="auto"Used for buffering large files; it can take one of three values:
    • "none" does not buffer the file
    • "auto" buffers the media file
    • "metadata" buffers only the metadata for the file
  • poster="poster.png"The URL of an image which will be displayed before the video is played.

If you don't add the controls attribute, build your interface using the appropriate JavaScript API. At a minimum, the interface must include a way to start and stop the media, and to adjust the volume.

The <audio> element is exactly like the <video> element, except that it doesn't have the width, height and poster attributes.

Semantic math elements

Every text in MathML formulas must be included inside these container elements, called token elements.

  • mn: Number literal (2, 3.141, 0x0045, ...)
  • mo: Operator or other punctuation sign (+, ∑, [, ...)
  • mi: Identifier (x, cos, π, ...)
  • mtext: Other text (if, such as, not, ...)

Single-character <mi> elements are automatically italicized. To prevent this, use the mathvariant attribute.

<mi>sin</mi>                      <!-- Multi-character identifier, not italicized -->
<mtext>&nbsp;</mtext>
<mi>x</mi>                        <!-- Single-character identifier, italicized -->
<mtext>&nbsp</mtext>
<mi mathvariant="normal">x</mi>   <!-- Single-character identifier, explicitly not italicized -->

sin   x   x

The browser will look up any operator enclosed in <mo> in a operator dictionary so that the necessary spacing or stretching between surrounding elements is respected.

Style: CSS

Styles, properties and values

You change the look of an element by applying a style to it.

<p style="text-decoration: underline;">This element has a style applied to it.</p>

This element has a style applied to it.

A style is made out of declarations, each consisting of a property and a list of values.

<p style="text-decoration: underline;
          font-weight: bold;
          font-family: Georgia, serif;">The style applied to this element contains 3 declarations.</p>

The style applied to this element contains 3 declarations.

Each property expects values of a different type.

<p style="width: 400px;
          padding: 10px;
          background-color: pink;
          color: blue;">This paragraph has properties of type 'color' and properties of type 'length'.</p>

This paragraph has properties of type 'color' and properties of type 'length'.

A property can have fallback values in case the first one can't be applied.

<p style="font-family: Helvetica, Arial;">This paragraph is styled with a property that has more than 1 value.</p>

This paragraph is styled with a property that has more than 1 value.

In the example, the font of the paragraph is set to Helvetica, but that might not be available on the current system. In that case, Arial will take its place.

Some groups of properties have shorthands: a single property that combines multiple other ones.

<p style="font-style: italic;
          font-weight: bold;
          font-size: 0.8em;
          line-height: 1.2;
          font-family: Arial, sans-serif;">This paragraph doesn't use shorthand properties.</p>
<p style="font: italic bold 0.8em/1.2 Arial, sans-serif;">This paragraph does use shorthand properties.</p>

Rules

A CSS stylesheet is made of rules. Rules have the following syntax:

selector { property: value; }

A CSS rule opens with one or more selectors.

h1 { }    /* Selects all the elements of type <h1> */
p, li { } /* Selects all the elements of type <p> and all the elements of type <li> */

Inside the braces will be one or more declarations, which take the form of a property and a list of values.

h1 {
  color: red;     /* Sets the 'color' property to have the 'red' value */
  font-size: 3em; /* Sets the 'font-size' property to have the '3em' value */
}

CSS properties have different data types, and each data type specifies its values in a different way.

Selectors and combinators

Type selectors directly match an HTML element type - for example, all elements of type <p>.
They are also called tag name selectors or element selectors. Just like HTML tags, they are not case-sensitive.

p { /* Select all paragraphs */
  color: green;
}
<p>This is the first paragraph, and it's green.</p>
<p>This is the second paragraph, and it's green too.</p>

This is the first paragraph, and it's green.

This is the second paragraph, and it's green too.

To select a subset of the elements, use a class selector.

.special { /* Select all elements with class 'special' */
  color: orange;
  font-weight: bold;
}
<ul>
  <li>Item one</li>
  <li class="special">Item two</li> <!-- This element has a class of "special" -->
  <li>Item three</li>
</ul>
  • Item one
  • Item two
  • Item three

If you want to select only elements of a specific type and with a specific class, you can write the type and the class as a single selector.

li.special { /* Select all list items of class 'special' */
  color: orange;
  font-weight: bold;
}

Do not put a space between the type and the class. If you do it, it becomes a descendant combinator.

ID selectors target an element that has a specific value for its id attribute:

#cool { /* Select only the element with id 'cool' */
  font-family: Arial;
}
#cooler { /* Select only the element with id 'cooler' */
  font-family: Georgia;
}
<p><span id="cool">Sans-serif fonts are cool;</span> <span id="cooler">but serif fonts are cooler.</span></p>

Sans-serif fonts are cool; but serif fonts are cooler.

There should't be more than 1 element with the same id in a single page. It can cause problems for scripts trying to get references to elements.

If you want to select only the element of a specific type and with a specific id, you can write the type and the id as a single selector. 

p.sans { /* Select all paragraphs with id 'sans' */
  font-family: Arial;
}

Note: this doesn't make much sense, as a page should only have 1 element with a specific id; you can't get more specific than 1 element. I guess it is there for completeness sake, but it is never used.

You can select elements based on the presence of a certain attribute on an element using an attribute selector.

a[title] { /* Selects only elements with the attribute 'title' */
  font-weight: bold;
}
<p>I do not live in the <abbr>US</abbr> I live in the <abbr title="European Union">EU</abbr>.</p>

I do not live in the US; I live in the EU.

You can select elements based on the presence of an attribute with a particular value.

a[title="European Union"] { /* Selects only elements where the attribute 'title' has value 'European Union' */
  font-weight: bold;
}
<p>I do not live in the <abbr title="United States">US</abbr> I live in the <abbr title="European Union">EU</abbr>.</p>

I do not live in the US; I live in the EU.

You can select elements based on the presence of an attribute with a particular value among (eventually) others values.

a[title~="United"] { /* Selects only elements where the attribute 'title' and whose list of values contains 'United' */
  font-weight: bold;
}
<p>I do not live in the <abbr title="United States">US</abbr> neither do I live in the <abbr title="United Kingdom">UK</abbr>.</p>

I do not live in the US; neither do I live in the UK.

There is other syntax to select substrings of attribute values.

[attr|=value] Matches elements with an attr attribute whose value is exactly value or begins with value immediately followed by a hyphen (-).
[attr^=value] Matches elements with an attr attribute, whose value begins with value.
[attr$=value] Matches elements with an attr attribute whose value ends with value.
[attr*=value] Matches elements with an attr attribute whose value contains value anywhere within the string.

Some useful pseudo-classes are the following.

  • :empty: Represents an element with no children other than white-space characters.
  • :first-child: Match an element that is the first of its siblings. Also see :last-child.
  • :nth-child: Uses An+B notation to select elements from a list of sibling elements. Also see :nth-last-child.
  • :hover: Matches when a user designates an item with a pointing device, such as holding the mouse pointer over the item.
  • :active: Matches when an item is being activated by the user. For example, when the item is clicked on.
  • :focus: Matches when an element has focus.
  • :enabled: Represents a user interface element that is in an enabled state. Also see :disabled.
  • :blank: Matches a user-input element which is empty, containing an empty string or other null input.
  • :valid: Matches an element with valid contents. For example, an input element with the type 'email' that contains a validly formed email address or an empty value if the control is not required. Also see :invalid.
  • :in-range: Applies to elements with range limitations. For example, a slider control when the selected value is in the allowed range. Also see :out-of-range.
  • :link: Matches links that have not yet been visited. Also see :visited.
  • :playing: Represents a media element that is playing. Also see :paused.

There are more pseudo-classes not listed here. See a list of all the pseudo-classes.

Descendant combinators

To select only an <em> that is nested inside an <li> element, you can use a selector called the descendant combinator, which takes the form of a space between two other selectors:

Styling text

Text inside an element is laid out inside the element's content box. It starts at the top left of the content area (or the top right, in the case of RTL language content), and flows towards the end of the line.
Once it reaches the end, it goes down to the next line and flows to the end again.

The text inside an element is all affected as one single entity. You can't select and style subsections of text unless you wrap them in an appropriate element (such as a <span> or <strong>), or use a text-specific
pseudo-element like ::first-letter (selects the first letter of an element's text), ::first-line (selects the first line of an element's text), or ::selection (selects the text currently highlighted by the cursor).

Web fonts

CSS allows you to specify font files, available on the web, to be downloaded along with your website as it's accessed. This means that any browser supporting this CSS feature can display the fonts you've specifically chosen.

First of all, you have a @font-face ruleset at the start of the CSS, which specifies the font file(s) to download.

@font-face {
  font-family: "myFont";
  src: url("myFont.woff2");
}

Below this you use the font family name specified inside @font-face to apply your custom font to anything you like, as normal.

html {
  font-family: "myFont", "Bitstream Vera Serif", serif;
}

Note that:

  • Fonts generally aren't free to use. You have to pay for them and/or follow other license conditions, such as crediting the font creator in your code (or on your site).
  • All major browsers support WOFF/WOFF2.
  • WOFF2 supports the entirety of the TrueType and OpenType specifications, including variable fonts, chromatic fonts, and font collections.
  • The order in which you list font files is important. If you provide the browser with a list of multiple font files to download, the browser will choose the first font file it's able to use. That's why the format you list first should be the preferred format — that is, WOFF2 — with the older formats listed after that. Browsers that don't understand one format will then fall back to the next format in the list.
  • If you need to work with legacy browsers, you should provide EOT, TTF, and SVG web fonts for download.

Font stacks

Since you can't guarantee the availability of the fonts you want to use on your webpages (even a web font could fail for some reason), you can supply a font stack so that the browser has multiple fonts it can choose from. This involves a font-family value consisting of multiple font names separated by commas, e.g.,

p {
  font-family: "Trebuchet MS", Verdana, sans-serif;
}

In such a case, the browser starts at the beginning of the list and looks to see if that font is available on the machine. If it is, it applies that font to the selected elements. If not, it moves on to the next font, and so on.

It is a good idea to provide a suitable generic font name at the end of the stack so that if none of the listed fonts are available, the browser can at least provide something approximately suitable.

JavaScript linking

There are three ways to link a script to a webpage.

External JavaScript

External JavaScript is written in a separate file with the .js extension, and then referenced in the HTML using <script> tags:

<script src="script.js" defer></script>

The defer attribute tells the browser to parse the script but to not run it immediately, and instead to wait until the document tree has been generated.

Internal JavaScript

Internal JavaScript is written directly inside the HTML file, between two <script> tags:

<script>
  // JavaScript goes here
</script>

Inline JavaScript

Inline JavaScript is a snippet of code written as the value of an element's attribute:

<button onclick="/* JavaScript goes here */"></button>

Events

Events are "things that happen" which the system tells you about so your code can react to them.
There are many different types of events that can occur, for example:

  • The user selects, clicks, or hovers the cursor over a certain element.
  • The user presses a key on the keyboard.
  • The user resizes or closes the browser window.
  • A web page finishes loading.
  • A form is submitted.
  • A video is played, paused, or ends.
  • An error occurs.

To react to an event, you define a block of code that should run when the event fires - the event handler - and then
you register it to the element that will fire the event.

<button type="button">Randomize background</button> 
const button = document.querySelector("button"); /* Find the first button in the page */

button.addEventListener("click", randomize_background()) /* Add a listener to the button's "click" event */

function randomize_background() {
  /* When the button is clicked, run the following code */
  const color = `rgb(${random(192, 255)} ${random(192, 255)} ${random(192, 255)})`;
  document.body.style.backgroundColor = color;
});

function random(min, max) {
  return Math.floor(Math.random() * (max - min) + min);
}

Adding and removing events

There are two ways to add event handlers to elements:
by calling the element.addEventListener(event: String, action: (void) => void) method.

button.addEventListener("click", do_stuff);

or by assigning to the element.event_name property:

button.onclick = do_stuff;

The name of the property is always the same as the event prefixed by "on".

  • "click" => onclick
  • "dblclick" => ondblclick
  • "mouseover" => onmouseover
  • ...

If you call addEventListener multiple times for the same event, multiple handlers will run when the event is fired:

button.addEventListener("click", do_thing_A);
button.addEventListener("click", do_thing_B);

You can't obtain the same result if you instead add events by assigning to the element's properties, as the new handler will overwrite the old one:

button.onclick = do_thing_A;
button.onclick = do_thing_B;

If you've added an event handler using addEventListener(), you can remove it again using the removeEventListener() method.

button.removeEventListener("click", randomize_background);

Button events

A button has many kinds of events it can fire, such as "click", "mouseover", and "keydown".

The optional event parameter

The target field of this parameter refers to the HTML element that triggered the event (e.g. for a "click" event, event.target represents

Web performance

  • Reduce load time: Make your files as small as possible, reduce the number of HTTP requests made as much as possible, and employ clever loading techniques (such as preload)
  • Make the site usable as soon as possible: Load assets in a sensible order so that the user can start to actually use it really quickly.
  • Make it smooth and pleasurable: use CSS rather than JavaScript for animations, minimizing the number of repaints the UI requires due to changes in the DOM.
