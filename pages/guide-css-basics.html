Style: CSS

Styles, properties and values

You change the look of an element by applying a style to it.

<p style="text-decoration: underline;">This element has a style applied to it.</p>

This element has a style applied to it.

A style is made out of declarations, each consisting of a property and a list of values.

<p style="text-decoration: underline;
          font-weight: bold;
          font-family: Georgia, serif;">The style applied to this element contains 3 declarations.</p>

The style applied to this element contains 3 declarations.

Each property expects values of a different type.

<p style="width: 400px;
          padding: 10px;
          background-color: pink;
          color: blue;">This paragraph has properties of type ‘color’ and properties of type ‘length’.</p>

This paragraph has properties of type ‘color’ and properties of type ‘length’.

A property can have fallback values in case the first ones can't be applied.

<p style="font-family: Helvetica, Arial;">This paragraph's font is set to Helvetica, or Arial if Helvetica is not available on the current system.</p>

This paragraph's font is set to Helvetica, or Arial if Helvetica is not available on the current system.

Some groups of properties have shorthands, single properties that combine multiple other ones.

<p style="font: italic bold 1em/1.2 Arial, sans-serif;">This paragraph uses a shorthand property.</p>

This paragraph uses a shorthand property.

Each shorthand property accepts its values in a specific order. See details for each shorthand.

Rules and stylesheets

To apply a style to a single HTML element, modify its style attribute.

<p style="color: blue;">This paragraph uses inline styles.</p>

This paragraph uses inline styles.

To apply a style to multiple HTML elements at once, you can write a stylesheet containing rules. You can write rules between two <style> tags inside the <head> to define an internal stylesheet.

<head>
  <style>
    p { color: blue; }
  </style>
</head>
<body>
  <p>This page is styled with CSS written directly at the top of the document.</p>
</body>

This page is styled with CSS written directly at the top of the document.

You can also write rules in a separate file with the .css extension to define an external stylesheet. To reference it, put a <link> attribute in the <head> of the HTML document.

<head>
  <link rel="stylesheet" href="styles.css" />
</head>

Note: As I understand it, you should put the CSS in the <head> of the document so that the browser can parse it alongside the HTML and immediately generate the correct layout, rather than generating the wrong layout first and then changing it when styles are applied.

Stylesheet comments

Text enclosed within /* and */ is treated as a comment and will not change the behaviour of a stylesheet.

/* This is a comment! Boo-yaa */

The box model

Every HTML element is contained in a box.

The box around this paragraph has been made visible.

Some elements, such as paragraphs, have a box of type block.

This element is a paragraph.

This is another paragraph; as such, it is automatically placed on a new line, and it expands horizontally to fit the width of its parent (unless otherwise specified).

Other elements, such as anchors, have a box of type inline.

This paragraph contains a link. A link does not cause the text to go a new line, and its size matches the size of its content.

When a style is applied to an element, it is actually applied to its box.

<p style="background-color: #fdd;">This paragraph is short, but the whole box has been colored.</p>

This paragraph is short, but the whole box has been colored.

The box type of an element affects how it behaves in relationship with other elements.

Block Inline
Inserts a newline before and after. Does not insert newlines.
width and height properties are respected. If the width isn't specified, it will stretch to fill its parent box. width and height properties are ignored.
Padding, border and margin will push other elements away. Padding, border and margin will not push other elements away.

Parts of a box

A block box is made of (from inside to outside):

  • A content box.
  • A padding box.
  • A border box.
  • A margin box.

The padding property controls the space around the content and inside the border.

<p style="padding: 15px;">There is some space between this text and its border.</p>

There is some space between this text and its border.

The border property controls the border of a box.

<p style="border: 1px solid black;">The border around this paragraph is now black.</p>

The border around this paragraph is now black.

The margin property controls the space around the border.

<p style="margin: 20px;">There is some space around the border of this paragraph.</p>

There is some space around the border of this paragraph.

Standard and alternate box models

In the standard box model, properties like width, height, inline-size and block-size describe the size of the content box. In the alternate box model, they describe the total size of content, padding and border.

div {
  border: 1px solid #bbb;
  margin: 40px; padding: 40px;
  width: 300px; height: 150px;
}
<div>I use the standard box model.</div>
<div style="box-sizing: border-box;">I use the alternate box model.</div>
I use the standard box model.
I use the alternate box model.

These two boxes have the same width and height values, but the second one is set to use the alternate box model with the box-sizing property.

To use the alternate box model for all of your elements (which is a common choice among developers), see this article on box sizing.

Inline and block directions

Inline direction refers to the direction the text is flowing in. By default, the inline-size property means the same as width.

<p style="inline-size: 150px; padding: 5px;">This paragraph is very narrow. It has been squashed by setting its inline size to a small value.</p>

This paragraph is very narrow. It has been squashed by setting its inline size to a small value.

Block direction refers to direction blocks are displayed on a page. By default, the block-size property means the same as height.

<p style="block-size: 100px; padding: 5px;">This paragraph is very tall. It has been stretched vertically by setting its block size to a big value.</p>

This paragraph is very tall. It has been stretched vertically by setting its block size to a big value.

The inline and block size properties change their meaning depending on the text direction: horizontal (like most languages) or vertical (like Japanese). The text direction can be set with the writing-mode property:

<p style="writing-mode: vertical-rl; inline-size: 90px; padding: 5px; font-size: 1.2em;">Sideways text.</p>

Sideways text.

In this case, inline-size means the same as height.

The width and height properties are called physical properties; the inline-size and block-size properties are called logical properties.

Selectors and combinators

A CSS rule starts with one or more selectors.

em, strong { }

Type selectors

Type selectors directly match an HTML element type. They are also called tag name selectors or element selectors. Just like HTML tags, they are not case-sensitive.

P { /* Select all paragraphs */
  color: green;
}
<p>This is the first paragraph, and it's green.</p>
<p>This is the second paragraph, and it's green too.</p>

This is the first paragraph, and it's green.

This is the second paragraph, and it's green too.

Class selectors

To select a subset of the elements, use a class selector.

.special { /* Select all elements with class 'special' */
  color: orange;
  font-weight: bold;
}
<ul>
  <li>Item one</li>
  <li class="special">Item two</li> <!-- This list item has a class of 'special' -->
  <li>Item three</li>
</ul>
</p class="special">Paragraph</p>   <!-- This paragraph has a class of 'special' -->
  • Item one
  • Item two
  • Item three

Paragraph.

If you want to select only elements of a specific type and with a specific class, you can write the type and the class as a single selector.

li.special { /* Select all list items of class 'special' */
  color: orange;
  font-weight: bold;
}
<ul>
  <li>Item one</li>
  <li class="special">Item two</li> <!-- This list item has a class of 'special' and is selected by the rule -->
  <li>Item three</li>
</ul>
</p class="special">Paragraph</p>   <!-- This paragraph has a class of 'special' and is NOT selected by the rule -->
  • Item one
  • Item two
  • Item three

Paragraph.

Do not put a space between the type and the class. If you do, it becomes a descendant combinator.

Id selectors

To target exactly 1 element, use an ID selector.

#cool {      /* Select only the element with id 'cool' */
  font-family: Arial;
}
#cooler {    /* Select only the element with id 'cooler' */
  font-family: Georgia;
}
<p>
  <span id="cool">Sans-serif fonts are cool;</span>      <!-- This span has an id of 'cool' -->
  <span id="cooler">but serif fonts are cooler.</span>   <!-- This span has an id of 'cooler' -->
</p>

Sans-serif fonts are cool; but serif fonts are cooler.

There should't be more than 1 element with the same id in a single page. It can cause problems for scripts trying to get references to elements.

If you want to select only the element of a specific type and with a specific id, you can write the type and the id as a single selector. 

p#sans { /* Select all paragraphs with id 'sans' */
  font-family: Arial;
}

Note: this doesn't make much sense, as a page should only have 1 element with a specific id; you can't get more specific than 1 element. I guess it is there for completeness sake, but it is never used.

Attribute selectors

You can select elements based on the presence of a certain attribute on an element using an attribute selector.

a[title] { /* Selects only elements with the attribute 'title' */
  font-weight: bold;
}
<p>I do not live in the <abbr>US</abbr>; I live in the <abbr title="European Union">EU</abbr>.</p>

I do not live in the US; I live in the EU.

You can select elements based on the presence of an attribute with a particular value.

a[title="European Union"] { /* Selects only elements where the attribute 'title' has value 'European Union' */
  font-weight: bold;
}
<p>I do not live in the <abbr title="United States">US</abbr>; I live in the <abbr title="European Union">EU</abbr>.</p>

I do not live in the US; I live in the EU.

You can select elements based on the presence of an attribute with a particular value among (eventually) others values.

a[title~="United"] { /* Selects only elements where the attribute 'title' and whose list of values contains 'United' */
  font-weight: bold;
}
<p>I do not live in the <abbr title="United States">US</abbr>; neither do I live in the <abbr title="United Kingdom">UK</abbr>.</p>

I do not live in the US; neither do I live in the UK.

There is other syntax to select substrings of attribute values.

[attr|=value] Matches elements with an attr attribute whose value is exactly value or begins with value immediately followed by a hyphen (-).
[attr^=value] Matches elements with an attr attribute, whose value begins with value.
[attr$=value] Matches elements with an attr attribute whose value ends with value.
[attr*=value] Matches elements with an attr attribute whose value contains value anywhere within the string.

Pseudo-class selectors

Pseudo-class selectors (or state selectors) select style certain states of an element. For example, they are the first element of their type (:first-child), or they are being hovered over by the mouse pointer (:hover).

a:hover { /* Selects links which are being hovered over */
  color: pink;
  background-color: black;
}
<a>Hover over me!</a>

Hover over me!

Some useful pseudo-classes are the following.

  • :empty: Represents an element with no children other than white-space characters.
  • :first-child: Match an element that is the first of its siblings. Also see :last-child.
  • :nth-child: Uses An+B notation to select elements from a list of sibling elements. Also see :nth-last-child.
  • :hover: Matches when a user designates an item with a pointing device, such as holding the mouse pointer over the item.
  • :active: Matches when an item is being activated by the user. For example, when the item is clicked on.
  • :focus: Matches when an element has focus.
  • :enabled: Represents a user interface element that is in an enabled state. Also see :disabled.
  • :blank: Matches a user-input element which is empty, containing an empty string or other null input.
  • :valid: Matches an element with valid contents. For example, an input element with the type 'email' that contains a validly formed email address or an empty value if the control is not required. Also see :invalid.
  • :in-range: Applies to elements with range limitations. For example, a slider control when the selected value is in the allowed range. Also see :out-of-range.
  • :link: Matches links that have not yet been visited. Also see :visited.
  • :playing: Represents a media element that is playing. Also see :paused.

There are more pseudo-classes not listed here. See a list of all the pseudo-classes.

Pseudo-element selectors

Pseudo-element selectors select a certain part of an element rather than the element itself.

p::first-line {
    text-decoration: underline;
}
<p>The first line of the paragraph is underlined,<br /> while every other one isn't.</p>

The first line of the paragraph is underlined,
while every other one isn't.

Descendant combinators

To restrict the selection to the subtree of an element, put a space between two selectors to create a descendant combinator.

li em { /* Select any <em> element that is inside (a descendant of) a <li>. */
  color: rebeccapurple;
}

Child combinators

To restrict the selection to the direct children of an element, put a > between two selectors to create a child combinator.

article > p { /* Select any <p> element that is a child of an <article>. */
  font-family: Arial;
}

Next-sibling combinators

To restrict the selection to the first element on the same level of another, put a + between two selectors to create a next-sibling combinator.

h1 + p { /* Select any <p> element that is immediately after (a next-sibling of) an <h1>. */
  font-size: 200%;
}

Styling text

Text inside an element is laid out inside the element's content box. It starts at the top left of the content area (or the top right, in the case of RTL language content), and flows towards the end of the line.
Once it reaches the end, it goes down to the next line and flows to the end again.

The text inside an element is all affected as one single entity. You can't select and style subsections of text unless you wrap them in an appropriate element (such as a <span> or <strong>), or use a text-specific
pseudo-element like ::first-letter (selects the first letter of an element's text), ::first-line (selects the first line of an element's text), or ::selection (selects the text currently highlighted by the cursor).

Color

The color property sets the color of the foreground content of the selected elements, which is usually the text, but can also include a couple of other things, such as an underline or overline placed on text using the text-decoration property. Color can accept any CSS color unit.

p { color: darkred; }
<p>Tommy the cat had many a story to tell.</p>

Tommy the cat had many a story to tell.

Font

The font-family property allows you to specify a font (or list of fonts) to apply to the selected elements.

p { font-family: Helvetica, Arial; }
<p>Tommy the cat had many a story to tell.</p>

Tommy the cat had many a story to tell.

Default fonts

CSS defines five default names for fonts - the exact font face used from these generic names can vary between each browser and each operating system. It represents a worst case scenario where the browser will try its best to provide a font that looks appropriate.

  • serif: Fonts that have serifs.
  • sans-serif: Fonts that don't have serifs.
  • monospace: Fonts where every character has the same width, typically used in code listings.
  • cursive: Fonts that are intended to emulate handwriting, with flowing, connected strokes.
  • fantasy: Fonts that are intended to be decorative.

System fonts

Fonts provided by the operating system are generally available across all platforms and are therefore called web-safe fonts. The list of actual web safe fonts will change as operating systems evolve, but it's reasonable to consider the following fonts web safe, at least for now.

Web fonts

Font files can be downloaded along with your website as it's accessed. Fonts generally aren't free to use. You have to pay for them and/or follow other license conditions, such as crediting the font creator in your code or on your site.

Use a @font-face ruleset at the start of the stylesheet to specify the font file(s) to download. Below it, use the font family name as usual.

@font-face {
  font-family: "my-custom-font";
  src: url("my-custom-font.woff2");
}

html { font-family: "my-custom-font", serif; }

The font-family property defines the identifier through which the rest of the stylesheet can refer to the font. It can be any string, not necessarily the name of the font.

@font-face {
  font-family: "spongebob-squarepants";
  src: url("my-custom-font.woff2");
}

html { font-family: "spongebob-squarepants", serif; }

The src property expects the URL from which to load the font file, and an optional format value. Though optional, it is recommented so that browsers don't have to figure out the format themselves.
Multiple values can be listed here, so that older browsers can fallback to older formats they can recognize.

@font-face {
  font-family: "my-custom-font";
  src: url("my-custom-font.woff2") format("woff2"),
       url("my-custom-font.woff") format("woff");
}

To load multiple variations of the same font and assign them to a single identifier, see this article.

Note that:

  • All major browsers support WOFF/WOFF2.
  • WOFF2 supports the entirety of the TrueType and OpenType specifications, including variable fonts, chromatic fonts, and font collections.
  • The order in which you list font files is important. If you provide the browser with a list of multiple font files to download, the browser will choose the first font file it's able to use. That's why the format you list first should be the preferred format — that is, WOFF2 — with the older formats listed after that. Browsers that don't understand one format will then fall back to the next format in the list.
  • If you need to work with legacy browsers, you should provide EOT, TTF, and SVG web fonts for download.

Steps for using a web font from a font provider: 1) download the TTF/OTF file. 2) Use a converter to generate WOFF/WOFF2 files + the corresponding stylesheet containint the @font-face rule.

Font stacks

Since you can't guarantee the availability of the fonts you want to use on your webpages (even a web font could fail for some reason), you can supply a font stack so that the browser has multiple fonts it can choose from.

body { font-family: "Trebuchet MS", Verdana, sans-serif; }

In such a case, the browser starts at the beginning of the list and looks to see if that font is available on the machine. If it is, it applies that font to the selected elements. If not, it moves on to the next font, and so on. It is a good idea to provide a suitable generic font name at the end of the stack so that if none of the listed fonts are available, the browser can at least provide something approximately suitable.

Styling forms

Simple form controls, like text, can be selected and styled by using attribute selectors.

input[type="text"] { border: 1px solid black; }
<input type="text" id="name" />

Many of the more complex input types are rendered by the operating system and are inaccessible to styling, so they are probabily look different for different users.

In some browsers, form elements do not inherit font styling by default. Therefore, if you want to be sure that your form fields use the font defined on the body, or on a parent element, you should add this rule to your CSS.

button,
input,
select,
textarea {
  font-family: inherit;
  font-size: 100%;
}

Across browsers, form elements use different box sizing rules for different widgets. It is also a good idea to set margins and padding to 0 on all elements, then add these back in when styling particular controls.

button,
input,
select,
textarea {
  box-sizing: border-box;
  padding: 0;
  margin: 0;
}

Set overflow: auto on <textarea> elements to stop showing a scrollbar when there is no need for one.

textarea { overflow: auto; }

CSS layout

There are many ways to change how elements are laid out on a page. Many of them have to do with the display property, while others involve other properties.

Normal flow

By default, the HTML is displayed in the exact order in which it appears in the source code, with block elements stacked on top of one another and inline elements beside one another.

I love my cat.

  • Buy cat food;
  • Exercise;
  • Cheer up friend;

The end!

By default, block-level elements are laid out in the block flow direction, which is based on the parent's writing mode (initial: horizontal-tb). Each element will appear on a new line below the last one, with each one separated by whatever margin that's been specified. In English, for example, (or any other horizontal, top to bottom writing mode) block-level elements are laid out vertically.

Inline elements behave differently. They don't appear on new lines; instead, they all sit on the same line along with any adjacent (or wrapped) text content as long as there is space for them to do so inside the width of the parent block level element. If there isn't space, then the overflowing content will move down to a new line.

Margin collapsing

If two vertically adjacent elements both have a margin set on them and their margins touch, the larger of the two margins remains and the smaller one disappears. This is only relevant in the vertical direction.

p {
  margin: 10px;
}

These paragraphs are separated by the width of their margins.

Because of margin collapsing, they are separated by the width of one of their margins, not both.

Custom flow

The default box type of an element can be changed using the display property.

<p style="display: inline;">This is the first paragraph.</p>
<p style="display: inline;">This is the second paragraph, and it is displayed on the same line as the previous one.</p>

This is the first paragraph.

This is the second paragraph, and it is displayed on the same line as the previous one.

<p id="block_link">This paragraph contains a <a style="display: block;" href="#block_link">link</a> which is treated as a block.</p>

This paragraph contains a link which is treated as a block.

To change how an element interacts with the elements around it, set its display property to one of these values:

Block Inline Inline-block
Inserts a newline before and after. Does not insert newlines. Does not insert newlines.
width and height properties are respected. If the width isn't specified, it will stretch to fill its parent box. width and height properties are ignored. width and height properties are respected.
Padding, border and margin will push other elements away. Padding, border and margin will not push other elements away. Padding, border and margin will push other elements away.

The display property also controls how an element's childred are laid out. More on this later.

The special display values flex and grid can be used to arrange elements more easily, either in one dimension (as a row or a column) or two dimensions (as a table) respectively.

Table-like layout

To position non-table elements in a tabular manner, use again the display property.

form {
  display: table; /* Treat forms as tables */
  margin: 0 auto;
}

form div {
  display: table-row; /* Treat 'div's as table rows */
}

form label,
form input {
  display: table-cell; /* Treat labels and inputs as table cells */
  margin-bottom: 10px;
}

form label {
  width: 200px;
  padding-right: 5%;
  text-align: right;
}

form input {
  width: 300px;
}

form p {
  display: table-caption; /* Treat paragraphs as table captions */
  caption-side: bottom; /* The caption property applies, even though it's technically just a paragraph */
  width: 300px;
  color: #999;
  font-style: italic;
}

Orbiting this at a distance of roughly ninety-two million miles is an utterly insignificant little blue green planet whose ape-descended life forms are so amazingly primitive that they still think digital watches are a pretty neat idea.

This planet has - or rather had - a problem, which was this: most of the people on it were unhappy for pretty much of the time. Many solutions were suggested for this problem, but most of these were largely concerned with the movements of small green pieces of paper, which is odd because on the whole it wasn't the small green pieces of paper that were unhappy.

And so the problem remained; lots of the people were mean, and most of them were miserable, even the ones with digital watches.

Many were increasingly of the opinion that they'd all made a big mistake in coming down from the trees in the first place. And some said that even the trees had been a bad move, and that no one should ever have left the oceans.

And then, one Thursday, nearly two thousand years after one man had been nailed to a tree for saying how great it would be to be nice to people for a change, one girl sitting on her own in a small cafe in Rickmansworth suddenly realized what it was that had been going wrong all this time, and she finally knew how the world could be made a good and happy place. This time it was right, it would work, and no one would have to get nailed to anything.

Sadly, however, before she could get to a phone to tell anyone about it, a terribly stupid catastrophe occurred, and the idea was lost forever.

This is not her story.

— Douglas Adams, Hitchhiker's Guide to the Galaxy
