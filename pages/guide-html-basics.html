HTML Basics

An HTML document is composed of a series of elements.

Tags

HTML elements are enclosed within tags.

<p>My cat is very grumpy.</p>

My cat is very grumpy.

Tag names are case-insensitive.

<p>My cat is very grumpy.</P>

My cat is very grumpy.

Tags can be nested.

<p>My cat is <em>very</em> grumpy.</p>

My cat is very grumpy.

Some tags are void and don't enclose anything. They stand on their own and don't need a corresponding closing tag.

<p>My cat<br />is very grumpy.</p>

My cat
is very grumpy.

Attributes

Elements can also have attributes.

<p style="color: darkred">My cat is very grumpy.</p>

My cat is very grumpy.

Attributes with only one value don't require the value to be specified. They are called boolean attributes (either they exist or they don't).

<input type="text" disabled="disabled" />
<input type="text" disabled />
<input type="text" />

Comments

Content enclosed within <!-- and --> is ignored by the browser.

<!-- I am a comment! Yayaya -->

Character entities

Use character entities to display characters which would normally be reserved, treated specially or difficult to type on a keyword. Entities start with a & and end with a ;.

<p>2 &lt; 5</p>

2 < 5

Organizing HTML

Each .html file corresponds to a different HTML page. It is not possible, using the default HTML features, to compose multiple .html files into a single HTML page, or to split a single .html file into multiple HTML pages.

The only way to get this result is to use offline programs to concatenate or split text files.

Making HTML accessible

Don't use HTML elements to style a webpage – only use them to convey meaning. To customize the page visually, use CSS.

HTML performance

Use Page Speed Insights to measure different aspects of your website performance.

As far as I understand, HTML optimization is all about structuring the page so that it needs to receive and store as little data as possible.

  • Reduce the number of nodes in the DOM - ideally under 1400 total nodes, better if under 800 total nodes.
  • Minify HTML files - that is, remove all comments and unnecessary spaces so that they load faster: 
    <!-- Non-minified version (169 bytes): -->
<img src    = "images/black-star-128.png"
     alt    = "A black star on a transparent background"
     title  = "Black star"
     width  = "128"
     height = "128" />

<!-- Minified version (131 bytes): -->
<img src="images/black-star-128.png"alt="A black star on a transparent background"title="Black star"width="128"height="128">

    Note that, as you can see, the difference in size is very small.

  • Clean up your media library by deleting unnecessary images and other media files for faster average page load times.
  • Keep files small: prefer .svg to other image formats when possible. Use .jpg for photos and .png for screenshots and logos. Make sure the source images are of the appropriate size for the page: don't waste time loading large images only to shrink them later.
  • Add width and height attributes to images and all other embeds. Specify the actual source dimensions so the browser can reserve space for them. This helps avoiding Cumulative Layout Shift.
